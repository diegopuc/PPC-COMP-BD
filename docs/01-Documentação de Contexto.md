# Introdução

Texto descritivo com a visão geral do projeto abordado. Inclui o contexto do domínio escolhido, o problema relacionado à organização e gestão de dados, os objetivos do trabalho, a justificativa e o público-alvo do sistema.

Neste projeto, os alunos deverão identificar um problema do mundo real que envolva armazenamento, organização e recuperação de informações, propondo uma solução baseada em modelagem e implementação de um banco de dados relacional.

A solução deverá contemplar as etapas de modelagem conceitual (MER), modelo lógico, modelo físico, além da implementação do banco de dados e uma interface simples que permita executar consultas SQL.

## Problema

Neste momento você deve apresentar o problema relacionado à gestão de dados que o seu sistema deverá resolver. Ainda não é o momento de apresentar a solução ou detalhes da aplicação.

Descreva a situação atual do problema e por que ele ocorre. Em muitos casos, organizações possuem informações dispersas em planilhas, documentos ou sistemas pouco estruturados, o que dificulta:

- organização das informações  
- recuperação de dados  
- consistência dos registros  
- geração de relatórios ou consultas  

Apresente também o contexto em que os dados são utilizados, como por exemplo:

- uma empresa  
- uma clínica  
- uma escola  
- um sistema de biblioteca  
- um sistema de vendas  
- controle de estoque  
- gestão de eventos  

O foco neste momento é compreender o domínio do problema e identificar quais dados precisam ser organizados e relacionados.

O grupo pode utilizar técnicas de análise como brainstorming, levantamento inicial de informações ou Design Thinking, buscando compreender melhor o problema e os dados envolvidos.

## Objetivos

Aqui você deve descrever os objetivos do trabalho.

O objetivo geral deve indicar que o projeto consiste em desenvolver e implementar um banco de dados relacional capaz de organizar e gerenciar as informações relacionadas ao problema apresentado.

Além disso, apresente objetivos específicos que estejam relacionados às etapas de modelagem e implementação do banco de dados.

Exemplos de objetivos específicos:

- Identificar e descrever as entidades, atributos e relacionamentos do domínio do problema.  
- Construir o Modelo Entidade-Relacionamento (MER).  
- Converter o modelo conceitual para o modelo lógico relacional.  
- Definir o modelo físico do banco de dados, incluindo tipos de dados e restrições.  
- Implementar o banco de dados utilizando SQL.  
- Desenvolver uma interface simples para executar consultas SQL e manipular dados.

## Justificativa

Descreva a importância de resolver o problema apresentado a partir da organização adequada das informações em um banco de dados.

Muitos sistemas dependem de uma boa estrutura de dados para garantir:

- integridade das informações  
- redução de redundâncias  
- facilidade de consulta  
- suporte à tomada de decisão  

A modelagem de dados permite compreender melhor como as informações se relacionam dentro de um domínio, servindo como base para o desenvolvimento de sistemas mais organizados e confiáveis.

Nesta seção, o grupo pode justificar por que escolheu determinado domínio (por exemplo, biblioteca, clínica, sistema de vendas, etc.) e explicar por que a criação de um banco de dados estruturado pode ajudar a resolver o problema identificado.

Também podem ser utilizados dados coletados por meio de entrevistas, observação ou análise de sistemas existentes, com o objetivo de compreender melhor a necessidade de organização das informações.

## Público-Alvo

Descreva quem são as pessoas ou organizações que utilizariam um sistema baseado no banco de dados desenvolvido.

O objetivo não é detalhar a interface do sistema, mas compreender quem interage com os dados e como essas informações são utilizadas no contexto do problema.

Exemplos de possíveis perfis de usuários:

- funcionários administrativos  
- gerentes ou gestores  
- atendentes  
- professores ou alunos  
- clientes  
- operadores do sistema  

Procure descrever características gerais do público, como:

- nível de familiaridade com tecnologia  
- tipo de informação que precisam acessar  
- frequência de uso do sistema  
- responsabilidades relacionadas aos dados  

Se desejarem, os grupos podem complementar essa análise com personas ou identificação de stakeholders, buscando compreender melhor quem são os envolvidos no domínio do problema.
